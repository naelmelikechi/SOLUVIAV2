1 Principes

Page /dashboard — tableau de bord de pilotage. Consolide les KPIs de toutes les pages SOLUVIA.

📈

Temps reel + snapshots

KPIs calcules en temps reel. Snapshot automatique le 1er de chaque mois pour comparer les periodes.

🔔

Alertes actives

Bloc alertes + badges dans le menu + emails automatiques. Les problemes remontent sans chercher.

⚙

Configurable

Chaque KPI est masquable. Les metriques affichees sont personnalisables par utilisateur.

RegleDetail
Page/dashboard — accessible depuis la navigation principale. La liste projets reste sur /projets.
CDPVoit les KPIs de ses projets uniquement (RLS). Recoit les alertes de ses projets.
AdminVoit les KPIs globaux (tous les projets, tous les CDP). Recoit les alertes globales.
Batch mensuelSnapshot sauvegarde automatiquement le 1er de chaque mois. Permet comparaison M vs M-1, M-3, M-12.

6 Snapshot mensuel & comparaisons

Chaque 1er du mois, SOLUVIA sauvegarde automatiquement tous les KPIs. Permet de comparer les evolutions.

Mecanisme

RegleDetail
DeclencheurCRON Supabase Edge Function — le 1er de chaque mois a 00:01.
Donnees sauvegardeesValeur de chaque KPI (financier, operationnel, qualite) pour le mois ecoule. Scope : global + par projet + par CDP.
RetentionSnapshots conserves indefiniment. Pas de purge automatique.
ImmutabiliteUn snapshot ne peut pas etre modifie apres creation. Les donnees du mois sont figees.

Codes couleur evolution — Vert = amelioration (hausse production, baisse retards). Rouge = degradation (hausse retards, baisse production). La semantique est inversee pour les metriques negatives (retard, abandons).

📌

Export CSV / XLSX — Le tableau d'evolution et les KPIs sont exportables en CSV ou XLSX. Exporte les donnees du snapshot mensuel visible (periodes affichees, KPIs affiches).

7 Notifications email

Emails automatiques sur regles fixes predefinies. Pas de configuration utilisateur.

AlerteFrequenceDestinataireContenu
Temps non saisi Quotidien (18h) Le CDP concerne "Vous n'avez pas saisi de temps aujourd'hui sur X projets."
Factures en retard Hebdomadaire (lundi 9h) Le CDP + admin "X factures en retard pour un total de Y €. Voir le detail."
Fenetre de facturation Le 25 du mois Tous les CDP "La fenetre de facturation est ouverte. X echeances a valider."
Fin de fenetre Le 2 du mois (veille de fermeture) CDP avec echeances non validees "La fenetre de facturation ferme demain. X echeances non facturees."
Rapport mensuel Le 1er du mois Admin Resume du mois ecoule : KPIs principaux, evolutions, alertes. Base sur le snapshot mensuel.
📌

Implementation — CRON Supabase Edge Functions + service d'envoi email (Resend, SendGrid ou equivalent). Les emails utilisent un template HTML sobre aux couleurs SOLUVIA.

9 Permissions & RLS

Meme modele que le reste de SOLUVIA.

A

Admin

Dashboard global : KPIs agreges sur tous les projets et CDP. Alertes globales. Rapport mensuel par email.

C

CDP

Dashboard filtre : KPIs de ses projets uniquement (cdp_id ou backup_cdp_id). Alertes de ses projets. Badges de ses projets.

10 Modele de donnees

Table de snapshots pour l'historique mensuel des KPIs.

Table kpi_snapshots

ColonneTypeDetail
idUUIDCle primaire
moisDATEPremier jour du mois (ex: 2026-04-01)
type_kpiTEXTproduction, facture, encaisse, en_retard, projets_actifs, contrats_actifs, apprenants_actifs, taux_saisie_temps, ...
valeurNUMERIC(14,2)Valeur du KPI au moment du snapshot
scopeTEXTglobal | projet | cdp
scope_idUUID NULLID du projet ou du CDP (NULL si scope = global)
created_atTIMESTAMPTZDate de creation du snapshot

Contraintes

ContrainteDetail
UniciteUNIQUE(mois, type_kpi, scope, scope_id) — un seul snapshot par KPI par scope par mois.
ImmutabilitePas de UPDATE autorise. Les snapshots sont en ecriture seule.
CRONEdge Function declenchee le 1er de chaque mois. Insere une ligne par KPI × scope.
RLSCDP : lecture des snapshots ou scope = 'global' ou scope_id = auth.uid() ou scope_id est un projet du CDP. Admin : tout.
