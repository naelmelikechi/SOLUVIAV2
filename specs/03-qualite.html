v1.0 Avril 2026 Spec 03 / 11

1 Vue d'ensemble

La page Qualite affiche l'etat des livrables Qualiopi pour chaque projet. 100% lecture seule — les modifications se font dans Eduvia.

Source des donnees

  • Toutes les donnees viennent de Eduvia
  • API qualite a venir dans Eduvia (pas encore disponible)
  • Synchronisation periodique (CRON)
  • Aucune modification possible dans SOLUVIA

Logique de conformite

  • Un livrable = une tache (fait / pas fait)
  • Une famille = conforme si 100% des livrables sont faits
  • Statut calcule automatiquement, pas de saisie manuelle
  • Clic sur un livrable → ouvre Eduvia dans un nouvel onglet

6 Donnees et synchronisation

Schema de stockage des donnees qualite dans Supabase.

Table qualite_familles

ColonneTypeDetail
idUUIDCle primaire
codeTEXT UNIQUEC1, C2... ADM, HQ, RGPD
libelleTEXTNom complet de la famille
descriptionTEXTDescription courte
nb_indicateursINTEGERNombre d'indicateurs dans la famille

Table qualite_livrables

ColonneTypeDetail
idUUIDCle primaire
projet_idUUID FKRef vers projets.id
famille_idUUID FKRef vers qualite_familles.id
eduvia_idINTEGERID du livrable dans Eduvia (pour la sync)
libelleTEXTNom du livrable
faitBOOLEANtrue = realise, false = a faire
last_sync_atTIMESTAMPTZDerniere synchronisation

Regles de synchronisation

  • API Eduvia qualite (a venir) sera appelee via Edge Function CRON
  • Match sur eduvia_id pour mise a jour (UPSERT)
  • Si un livrable disparait de l'API → archive = true, pas de suppression
  • Le statut conforme/non conforme est calcule en temps reel (pas stocke) : COUNT(fait = true) = COUNT(*) par famille et projet
