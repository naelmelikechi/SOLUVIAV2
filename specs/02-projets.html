Progression theorique — Calcul lineaire : temps ecoule depuis le debut du contrat ÷ duree totale × 100. Si la progression reelle est inferieure a la theorique, l'apprenant est en retard (affiche en orange ou rouge).

📌

Statuts contrat Eduvia — Le champ contract_state d'Eduvia peut prendre les valeurs suivantes : Actif contrat en cours, Suspendu temporairement interrompu, Resilie rupture anticipee, Termine arrive a echeance. SOLUVIA affiche le statut tel que recu d'Eduvia (lecture seule).

📌

Prise en charge — Montant du financement OPCO par contrat. Donnee prevue dans l'API Eduvia (champ a venir). En attendant, le champ sera vide ou saisi manuellement.

⚠

Cas limites — Projet sans contrat : le projet apparait dans la liste avec 0 apprentis, 0 taches, 0 factures. Les sections Finance et Contrats du detail affichent "Aucun contrat synchronise". Client sans cle API Eduvia : le client peut etre cree manuellement par l'admin (raison sociale, SIRET via Pappers). Les contrats et apprenants restent vides jusqu'a ajout d'une cle valide. Projet avec 0 apprenant : affichage normal, compteur a 0. Pas de traitement special.

Formulaire de creation

ChampTypeDetail
ClientAuto / SelectionPre-rempli depuis l'import Eduvia. Sinon selection parmi les clients existants.
TypologieSelectionListe dynamique : APP, POE, PDC... (geree par admin dans Parametres).
CDP assigneSelectionListe des utilisateurs avec role CDP.
Backup CDPSelectionListe des utilisateurs avec role CDP (different du CDP principal).
Date de debutDateDate libre saisie par l'admin.
Taux de commissionNombre (%)Pourcentage applique au CA OPCO pour calculer le revenu SOLUVIA.
StatutSelectionActif, En pause, Archive, Termine. Defaut : Actif.
⚠

Nom du projet = Ref — Le projet n'a pas de champ "nom" libre. Son identifiant (0042-DUP-APP) est genere automatiquement et sert de nom. Format : numero sequentiel + trigramme client + code typologie (voir Spec 01).

Statuts projet

Actif

Projet en cours. Visible par les CDP assignes.

En pause

Projet suspendu temporairement.

Termine

Projet fini. Reste visible en lecture seule.

Archive

Projet range. Opacite reduite dans les listes.

  • Seul l'admin peut changer le statut d'un projet
  • Un projet archive reste dans la base, jamais supprime
  • Un CDP peut toujours voir un projet termine/archive s'il y etait assigne
