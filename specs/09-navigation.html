2 Section Administration

Separee du menu principal par un trait. 3 sous-pages accessibles par admin (lecture+ecriture) et CDP (lecture seule).

PageURLAdminCDP
Clients/admin/clientsLecture + ecriture (CRUD)Lecture seule
Utilisateurs/admin/utilisateursLecture + ecriture (CRUD)Lecture seule

⚠

Parametres masques pour CDP — La page Parametres n'apparait pas dans le menu pour les CDP. Seuls Clients et Utilisateurs sont visibles en lecture seule. Cela evite de montrer des informations sensibles (cles API, coordonnees comptables).

6 Responsive & comportement global

Regles d'interface communes a toutes les pages de SOLUVIA.

BreakpointSidebarContenu
Desktop (≥ 1024px)Sidebar complete visible (240px). Toujours ouverte.Contenu a droite de la sidebar.
Tablette (768-1023px)Sidebar reduite : icones uniquement (60px). Hover pour voir le libelle.Contenu occupe le reste.
Mobile (< 768px)Sidebar masquee. Bouton hamburger en haut a gauche pour l'ouvrir (overlay).Contenu pleine largeur.

Elements communs

ElementDetail
BreadcrumbFil d'Ariane sous le header de chaque page. Cliquable. Ex: Administration > Clients > Heol Formation.
Header de pageTitre de la page + actions contextuelles (boutons, filtres). Toujours en haut.
TableauxPattern global : tri, filtre, recherche, colonnes configurables (spec 02).
ScrollSidebar fixe. Contenu scrollable. Header de page fixe en haut.
ThemeTheme clair uniquement (pas de dark mode pour l'instant). Palette verte SOLUVIA.
